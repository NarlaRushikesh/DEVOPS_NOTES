# Dynamic Resource Management Using Facts & Templates

## 1. Theory

### What Are Facts?
Facts are system information collected by the Puppet Agent using **Facter**.  
Examples:
- Hostname  
- OS Name  
- IP Address  
- Memory  
- CPU  
- Architecture  

They are automatically sent to the Puppet Master when the agent runs.

### What Are Templates?
Templates are dynamic files created using `.erb` (Embedded Ruby).  
They allow inserting variables and facts into configuration files.

Example inside a template:     
Hostname: <%= @facts['networking']['hostname'] %>


### Why Use Facts + Templates?
- To generate dynamic configs based on system details  
- To avoid hardcoding values  
- To create reusable, intelligent modules  
- To support different OS / hostnames / networks automatically

## 2. Practical Steps

### **Machine : Master**
```sh
cd /etc/puppetlabs/code/environments/production/modules/
```

### **Machine : Master**
```sh
sudo mkdir -p dynamicmodule/manifests
sudo mkdir -p dynamicmodule/templates
```

### **Machine : Master**
```sh
sudo nano dynamicmodule/templates/systeminfo.erb
```

Paste the following template:
```erb
System Information (Generated by Puppet Template)

Hostname: <%= @facts['networking']['hostname'] %>
Operating System: <%= @facts['os']['name'] %>
IP Address: <%= @facts['networking']['ip'] %>

This file was created dynamically using Puppet.
```

### **Machine : Master**
```sh
sudo nano dynamicmodule/manifests/init.pp
```

Paste the following:
```puppet
class dynamicmodule {

  file { '/tmp/system_info.txt':
    ensure  => file,
    content => template('dynamicmodule/systeminfo.erb'),
  }

  notify { "Dynamic module applied using facts and template": }
}
```

### **Machine : Master**
```sh
sudo nano /etc/puppetlabs/code/environments/production/manifests/site.pp
```

Add the following:
```puppet
node 'puppetagent' {
  include dynamicmodule
}
```

### **Machine : Agent**
```sh
sudo /opt/puppetlabs/bin/puppet agent -t
```

Expected Output:
```txt
Notice: /tmp/system_info.txt created
Notice: Dynamic module applied using facts and template
```

### **To verify file on agent**
```sh
cat /tmp/system_info.txt
```

Expected Outcome:
```txt
System Information (Generated by Puppet Template)

Hostname: puppetagent
Operating System: Ubuntu
IP Address: <Agent-IP>

This file was created dynamically using Puppet.
```
